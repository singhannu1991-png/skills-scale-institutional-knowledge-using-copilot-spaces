# Role-Based Onboarding Checklist

Use this checklist to help new team members understand their role, key responsibilities, and how they interact with other roles in OctoAcme projects.

---

## Before Your First Day

- [ ] Review [OctoAcme Project Management Overview](octoacme-project-management-overview.md)
- [ ] Understand your role definition in [OctoAcme Personas](octoacme-roles-and-personas.md)
- [ ] Read the [Roles Interaction Matrix](roles-interaction-matrix.md) to see how you collaborate with others
- [ ] Familiarize yourself with key OctoAcme processes relevant to your role (see sections below)

---

## Your First Week

### All Roles
- [ ] Attend project kickoff or team onboarding meeting
- [ ] Introduce yourself to all team members
- [ ] Get added to project communication channels (Slack, email lists, etc.)
- [ ] Understand the project's business goals, success metrics, and timeline
- [ ] Get access to project artifacts (project board, documentation, repos)

### Developers
- [ ] Clone project repository and set up local dev environment
- [ ] Review [Project Planning](octoacme-project-planning.md) for backlog structure
- [ ] Understand Definition of Done (DoD) and PR requirements
- [ ] Attend daily standups and sprint planning
- [ ] Pick up first story and get familiar with codebase

### Product Managers
- [ ] Review existing roadmap and prioritization criteria
- [ ] Understand success metrics and how they're tracked
- [ ] Schedule 1:1 with Project Manager and Technical Lead
- [ ] Review acceptance criteria template and prioritization process
- [ ] Begin backlog refinement if sprints are in progress

### Project Managers
- [ ] Review project charter and key milestones
- [ ] Access and understand current project board and status
- [ ] Schedule 1:1 with Product Manager and Scrum Master (if applicable)
- [ ] Understand risk register and escalation paths
- [ ] Set up communication cadence (standups, syncs, updates)

### QA / Testing Lead
- [ ] Review [Execution & Tracking](octoacme-execution-and-tracking.md) for quality standards
- [ ] Understand Definition of Done and acceptance criteria process
- [ ] Review existing test plans and automated test coverage
- [ ] Schedule 1:1 with Product Manager and developers
- [ ] Attend sprint planning and understand feature acceptance criteria

### Technical Lead / Architect
- [ ] Review system architecture and design decisions (ADRs if available)
- [ ] Understand technology stack and key system constraints
- [ ] Review existing code and architecture patterns
- [ ] Schedule 1:1 with developers and DevOps engineer
- [ ] Plan first architecture review or design session

### Designer / UX Lead
- [ ] Review existing design system and component library
- [ ] Understand user personas and key user flows
- [ ] Schedule 1:1 with Product Manager and developers
- [ ] Review accessibility and usability standards (WCAG, etc.)
- [ ] Attend design review ceremony (or schedule first one)

### DevOps / Infrastructure Engineer
- [ ] Review CI/CD pipelines and deployment processes
- [ ] Understand infrastructure setup (staging, production, etc.)
- [ ] Review [Release & Deployment Guide](octoacme-release-and-deployment.md)
- [ ] Schedule 1:1 with Technical Lead and developers
- [ ] Get familiar with monitoring, alerting, and observability setup

### Scrum Master / Agile Coach
- [ ] Review current sprint and team velocity
- [ ] Understand sprint ceremonies and their purpose
- [ ] Schedule 1:1 with Project Manager and team members
- [ ] Review retrospective notes and ongoing improvement items
- [ ] Attend next sprint planning, review, and retrospective

### Stakeholder / Sponsor
- [ ] Review project charter and business objectives
- [ ] Understand success metrics and reporting cadence
- [ ] Schedule alignment meeting with Project Manager and Product Manager
- [ ] Review risk register and key dependencies
- [ ] Attend project kick-off or executive briefing

---

## Your First Sprint / Month

### All Roles
- [ ] Participate fully in sprint ceremonies (planning, standups, review, retro)
- [ ] Get comfortable with project communication patterns and tools
- [ ] Ask questions and clarify role expectations
- [ ] Identify and flag early blockers or gaps
- [ ] Attend weekly stakeholder sync or status update

### Developers
- [ ] Complete first story and get feedback through PR review
- [ ] Contribute to sprint retrospective with feedback on processes
- [ ] Understand testing and deployment requirements
- [ ] Begin mentoring relationship with Technical Lead if needed
- [ ] Review lessons learned and best practices

### Product Managers
- [ ] Own first few backlog refinement sessions
- [ ] Validate acceptance criteria with QA and Tech Lead
- [ ] Attend weekly PM sync and stakeholder briefing
- [ ] Gather user feedback or research insights
- [ ] Refine roadmap based on team velocity and capacity

### Project Managers
- [ ] Run first sprint planning, standups, and retrospective
- [ ] Track sprint progress and identify risks early
- [ ] Conduct first risk register update
- [ ] Communicate status to stakeholders
- [ ] Begin working on any major dependencies or cross-team coordination

### QA / Testing Lead
- [ ] Execute first test cycle for in-progress features
- [ ] Define test strategy for next sprint features
- [ ] Collaborate with developers on test coverage
- [ ] Identify quality metrics to track
- [ ] Conduct pre-release validation for any completed features

### Technical Lead / Architect
- [ ] Conduct first design review with team
- [ ] Provide feedback on PR and architecture patterns
- [ ] Identify any technical risks or refactoring opportunities
- [ ] Mentor developers on best practices
- [ ] Document any architecture decisions (ADRs)

### Designer / UX Lead
- [ ] Conduct first design review or critique session
- [ ] Validate designs against product strategy
- [ ] Collaborate with developers on implementation fidelity
- [ ] Gather feedback on design system and component library
- [ ] Plan first usability research or validation session

### DevOps / Infrastructure Engineer
- [ ] Participate in first deployment (staging or prod)
- [ ] Review infrastructure requirements from developers and Tech Lead
- [ ] Validate CI/CD pipeline and alert/monitoring setup
- [ ] Document any improvements or automation opportunities
- [ ] Establish on-call handoff and incident response process

### Scrum Master / Agile Coach
- [ ] Facilitate first retrospective and capture action items
- [ ] Track and remove team impediments
- [ ] Coach team on sprint goal ownership
- [ ] Identify improvements to ceremonies or processes
- [ ] Celebrate team wins and psychological safety moments

### Stakeholder / Sponsor
- [ ] Attend first sprint review and retrospective
- [ ] Provide feedback on progress and direction
- [ ] Make any key trade-off or prioritization decisions
- [ ] Review business metrics and validate success
- [ ] Schedule next month's stakeholder briefing

---

## Ongoing Responsibilities

### Stay Connected
- [ ] Attend all relevant ceremonies (standups, planning, reviews, retros)
- [ ] Participate in weekly syncs with cross-functional partners
- [ ] Contribute to monthly stakeholder updates
- [ ] Review and provide feedback on process improvements

### Keep Learning
- [ ] Read relevant OctoAcme process docs as situations arise
- [ ] Ask questions and clarify role interactions
- [ ] Seek mentoring or coaching from senior team members
- [ ] Participate in knowledge-sharing sessions

### Contribute to Improvement
- [ ] Raise process gaps or inefficiencies in retrospectives
- [ ] Suggest enhancements to templates, checklists, or workflows
- [ ] Share best practices with team
- [ ] Use [Process Doc Update Issue Template](.github/ISSUE_TEMPLATE/add-update-content-to-process-docs.yml) to propose documentation updates
